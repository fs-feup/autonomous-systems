Detailed Description

Class responsible for recieving a set of uncolored cones and discovering their color This is done by performing a search that tries to find each side of the track by iteratively selecting the cone with the least cost to be added to each side of the track, according to the cost function defined in the class.

double ConeColoring::cost ( const Cone *  possible_next_cone,
int  n,
TrackSide  side 
) const
private

cost function used to minimize the cost of the path.

Based on the idea that, when there are at least two cones on one side of the track, the next cone is the one that minimizes a combination of the angle formed between consecutiv edges and the distance to the next cone.

Should not be called when current sides of the tracks are empty

Parameters
possible_next_conepossible next cone to be reached
nnumber of cones in the input vector
sidereach the cone form the left or right side
Returns
double cost of reaching the cone provided as a parameter

Returns
double estimate of the distance to the left side track
Note
assumming the car is at the initial_car_pose and going straight, the right side of the track can be estimated to be defined by: y - (car_y + 2*cos()) - tan()*(x - (car_x + 2*sen())) = 0 and the right side by : y - (car_y - 2*cos()) - tan()*(x - (car_x - 2*sen())) = 0 This also assumes the track width to be 4 meters since the minimum is 3 and the maximum is 5
